SpringBoot学习总结(一)--Hello World

SpringBoot SpringBoot 2018/07/17

如果有使用过 SSM 框架经验的同学肯定知道，在运行一个项目前你需要配置相当多的配置。复杂繁琐的配置让新手无所适从，而此时【此处应有掌声】SprignBoot 闪亮登场。

什么是 SpringBoot

Spring Boot 是由 Pivotal 团队提供的全新框架，其设计目的是用来简化 Spring 应用的初始搭建以及开发过程。该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置。简单来说，就是实现 约定大于配置 这一口号，你通过默认的设置就能正常启动一个项目啦！

快速入门

SpringBoot 创建有两种方式：

  • 访问 http://start.spring.io/ 的形式创建项目，然后导入到IDEA 中。
  • 直接在 IDEA 上创建项目 (实际上也是第一种方式，只是 IDEA 帮我们创建好了)

这里我们当然是直接在 IDEA 上创建项目咯！

SpringBoot 默认引入的模块有:

  • spring-boot-starter：核心模块，包括自动配置支持、日志和 YAML；

  • spring-boot-starter-test：测试模块，包括 JUnit、Hamcrest、Mockito。
