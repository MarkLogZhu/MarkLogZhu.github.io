后台管理系统实战(四)--用户登录实现

实战项目 Java Shiro SSM 2018/05/31

项目基本框架已经搭建好了，今天我们就来实现下用户登录功能。

逻辑说明

1.登录逻辑

  • 1.先从数据库中查找指定用户名的信息(前提是数据库中用户名唯一)
  • 2.判断密码是否一致
  • 3.判断状态是否正常
  • 4.将 token 写入到 cookie 中
  • 5.将用户信息写入到 redis 中
  • 6.返回登录成功

2.拦截器逻辑

  • 判断URL 是否是登录相关的URL
    • 是：放行
    • 否：判断用户是否登录
      • 未登录：跳转登录界面
      • 已登录：
        • 保存用户信息到 ThreadLocal
        • 刷新redis 用户信息过期时间
        • 放行

3.HttpInterceptor 逻辑

当请求完毕后将 ThreadLocal 中的用户等信息删除

AdminLTE

AdminLTE 是什么

AdminLTE 是基于 bootstrap3 的前端框架，并且将 bootstrap3 进行修改来适应自身的样式。

AdminLTE 也同样具备了后台应有模板，像登录模板、注册模板、个人模板、404模板、500模板、空白模板等，这些可以在下载后在 Examples 菜单里查看到。里面的组件 Widgets 页面，不要以为是一些像 easyui 的组件，其实也是 AdminLTE 定义的一些 css 样式，像 box 样式，是里面最基础最常用的一个。里面的 js 插件除了 bootstrap 自带那模态窗、导航、下拉菜单外，其它就是集成一些有名的第三方插件，像 select2、CK Editor、iCheck、DataTables 等等。
