Spring Security Spring Security 2019/05/03

除了表单身份认证之外，第三方登陆的场景也是比较常用的。第三方登陆需要使用到 OAuth 协议，还不太了解的小伙伴可以看下 OAuth协议

SpringSocial

SpringSocial 为我们封装了一个标准的 OAuth 协议的实现流程，我们来看下 SpringSocial 是如何封装的。

  • ServiceProvider 服务提供方

    • OAuth2Operations(OAuth2Template)，帮助开发者快速完成 OAuth2流程。

    • AbstractOAuth2ApiBinding，帮助开发者快速完成获取用户信息。

  • Connection 获取用户信息

    • Connection 即 OAuth2Connection 具体类。即第三方打交道获取用户信息。

    • Connection 是由 OAuth2ConnectionFactory 实现的。同时又依赖与服务提供方的认证与授权，每个服务提供方提供的用户信息类型，字段等不同，所以需要 ApiAdapter 来适配每一个服务提供方提供的信息。

  • 绑定用户与 Connection

    • Spring social 在数据库维护了一张表 UserConnection 表。将业务系统用户表中的 userId 与服务提供方用户 Connection 进行绑定关系。我们通过 UsersConnectionRepository 来对 UserConnection 进行增删改查。
