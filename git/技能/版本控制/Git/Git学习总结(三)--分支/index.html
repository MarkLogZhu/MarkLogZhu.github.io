分支管理策略

在合并分支时 Git 会默认使用 Fast forward 模式，在这种模式下，会丢掉分支提交的记录。可以通过
强制禁用 Fast forward 模式，Git 就会在 merge 时生成一个新的 commit，这样就能保留分支信息。

  • –no-ff 参数，表示禁用 Fast forward
  • -m 参数，把 commit 描述添加进去
