Spring Security 框架简介

Spring Security 是为基于 Spring 的应用程序提供声明式安全保护的安全性框架，它能够在 Web 请求级别和方法调用级别处理身份认证和授权。因为基于 Spring 框架，所以 Spring Security 充分利用了依赖注入和切面技术。

Spring Security 模块

Spring Security 被分为了11个模块，如下表所示：

模块 描述
ACL 支持通过访问控制列表 (access control list) 为域对象提供安全性
切面 当使用 Spring Security 注解时，会使用基于 AspectJ 的切面，而不是标准的 Spring AOP
cas 客户端 提供与 Jasig 的中心认证服务进行集成的功能
配置(Configuration) 包含通过 XML 和 Java 配置 Spring Security 的功能支持
核心(Core) 提供 Spring Security 基本库
加密 (Crptography) 提供了加密和解密的功能
LDAP 支持基于 LDAP 进行认证
OpenID 支持使用 OpenID 进行集中式认证
Remoting 提供了对 Spring Remoting 的支持
标签库(Tag Libray) Spring Security 的 JSP 标签库
Web 提供了 Spring Security 基于 Fiter 的 Web 安全性支持

配置 Spring Security

目前我们使用的都是 Spring Security 默认配置，如果需要自定义配置的话，我们需要一个实现 WebSecurityConfigurer 接口或者继承 WebSecurityConfigurerAdapter 的 Bean 对象。我们这边直接使用继承 WebSecurityConfigurerAdapter 的形式来配置。
我们 可以通过重载 WebSecurityConfigurerAdapter 的三个 configure() 方法来配置 Web 安全性：

方法 描述
configure(WebSecurity) 通过重载，配置Spring Security 的 Filter 链
configure(HttpSecurity) 通过重置，配置如何通过拦截器保护请求
configure(AuthenticationManagerBuilder) 通过重置，配置 user-detail 服务

通过 authorizeRequests() 和 anyRequest().authenticated() 配置要求所有的 HTTP 请求都需要进行认证。而 formLogin() 和 httpBasic() 设置就配置了应用支持表单登陆和 HTTP Basic 方式的认证。
