Tomcat学习总结(2)--Tomcat常用配置修改

tomcat tomcat 2018/04/11

修改项目端口号

使用场景: 多个 tomcat 同时运行
错误原因: tomcat 默认端口号是8080，同时存在多个 tomcat，端口会冲突
解决办法: 修改端口号

1) 进入 conf 目录下，编辑 server.xml 文件
2) 找到 Connector 节点并且协议是 http 的

<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443" />

3) 将端口从8080修改为其他端口，如8081

<Connector port="8081" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443" />

4) 重启 tomcat ,分别输入http://localhost:8080 和 http://localhost:8081

可以看到8080端口已经打不开页面了但是8081可以，说明端口修改成功


GET 请求乱码

使用场景: http 请求中文乱码
错误原因: 因为编码格式原因导致乱码
解决办法: 统一编码格式为 utf-8

1) 搭建测试环境

  • 进入 webapps 目录下，复制 root 目录并重命名为 test
  • 进入 test 目录，编辑 index.jsp 文件
    <!DOCTYPE html>
<html lang="en">
    <head>
           <meta charset="UTF-8" />    
    </head>
    <body>
        测试!
        <br>
        test!
    </body>
</html>
  • 重启 tomcat，浏览器中输入 http://localhost:8080/test/

2) 进入 conf 目录下，编辑 server.xml 文件
3) 找到 Connector 节点并且协议是 http 的

<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443" />

4) 增加编码格式

<Connector port="8080" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443" URIEncoding="UTF-8"/>

5) 重启 tomcat，浏览器中输入 http://localhost:8080/test/

修改默认项目名称

上面我们看到访问root项目时在地址栏不用输入 root ，但是访问 test 项目时就需要在地址栏中输入完整项目名称，让我们尝试修改下默认项目名

1) 进入 conf 目录下，编辑 server.xml 文件
2) 找到 Host 节点，并在节点内增加 Context 节点

<Host name="localhost"  appBase="webapps"
        unpackWARs="true" autoDeploy="true">
    <!-- 设置默认项目名称 -->
    <Context path="" docBase="D:\apache-tomcat-9.0.7\webapps\test" reloadable="true"/>     
   <Valve className="org.apache.catalina.valves.AccessLogValve" directory="logs"
           prefix="localhost_access_log" suffix=".txt"
           pattern="%h %l %u %t &quot;%r&quot; %s %b" />
</Host>

3) 重启 tomcat，浏览器中输入 http://localhost:8080

配置虚拟主机

　配置虚似主机就等同于配置一个网站。
　在Tomcat服务器配置一个虚拟主机(网站)，需要修改conf文件夹下的server.xml这个配置文件，使用Host元素进行配置，打开server.xml，可以看到Tomcat服务器自带的一个名称为localhost的虚拟主机(网站)，如下所示：

<Host name="localhost"  appBase="webapps"
      unpackWARs="true" autoDeploy="true">
   <Valve className="org.apache.catalina.valves.AccessLogValve" directory="logs"
         prefix="localhost_access_log" suffix=".txt"
         pattern="%h %l %u %t &quot;%r&quot; %s %b" />
</Host>

　　
　　平时我们将开发好的JavaWeb应用放到webapps文件夹下，然后就可以使用”http://localhost:端口号/JavaWebAppName"的方式去访问了，其实访问的就是name是"localhost"的那台虚拟主机(Host)，这台虚拟主机管理webapps文件夹下的所有web应用。
我们来自定义一个虚拟主机，
1) 进入 conf 目录下，编辑 server.xml 文件
2) 找到 Host 节点 ，在它下方新增一个节点

<Host name="marklogzhu"  appBase="webapps"
      unpackWARs="true" autoDeploy="true">   
</Host>

3) 打开C:\Windows\System32\drivers\etc下面的hosts文件 新增记录

127.0.0.1        marklogzhu

4) 重启tomcat，并输入 http://localhost:8080 、http://marklogzhu:8080 (注意：端口号，项目名称都已修改为默认)


可以看到两个地址都可以正常访问页面

修改Session有效时间

1) 进入 conf 目录下，编辑 web.xml 文件
2) 找到 session-config 节点，默认时间是30分钟

<session-config>
    <session-timeout>30</session-timeout>
</session-config>

3) 修改为你需要的时间即可
