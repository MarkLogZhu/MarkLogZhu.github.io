默认数据库连接池

细心的同学可能发现了，我们在配置的时候并没有设置数据库连接池，那就说明 SpringBoot 默认配置了一个数据库连接池，那么它是什么呢？我们来测试输出下看看：

启动，观察控制台输出：

可以发现 SpringBoot 1.5 默认采用的数据库连接池是 tomcat.jdbc。

除了 tomcat.jdbc 外，还有其他的数据库连接池吗？
答案当然是有的，如：

  • DBCP
  • C3P0
  • Druid
  • HikariCP

而在 Spring Boot 2.0 开始，默认的数据库连接池已经替换为 HikariCP了。

为何选择HikariCP

HiKariCP 是数据库连接池的一个后起之秀，号称性能最好，可以完美地 PK 掉其他连接池，是一个高性能的 JDBC 连接池，基于 BoneCP 做了不少的改进和优化。其作者还有另外一个开源作品——高性能的 JSON 解析器 HikariJSON。

它，超快，快到连 Spring Boot 2.0 都宣布支持了。

代码体积更是少的可怜，130kb。

我们来将默认 tomcat.jdbc 替换成 HikariCP。(同理，你也可以替换成自己喜欢的数据库连接池哦)
