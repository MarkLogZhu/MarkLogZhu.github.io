Maven学习总结(3)--POM

• 2017年12月12日

POM 是什么

POM 代表项目对象模型。它是 Maven 中工作的基本单位，这是一个 XML 文件。它始终保存在该项目基本目录中的 pom.xml 文件。

POM 包含的项目是使用 Maven 来构建的，它用来包含各种配置信息。

POM 也包含了目标和插件。在执行任务或目标时，Maven 会使用当前目录中的 POM。它读取 POM 得到所需要的配置信息，然后执行目标。

简单来说　setting.xml　是全局级别的配置文件，而　POM.xml 是项目级别的配置文件。

Maven 分为本地仓库和远程仓库

仓库的分类

本地仓库

本地仓库用来存储从远程仓库或中央仓库下载的插件和 jar 包，项目中使用的插件或 jar 包，优先从本地仓库查找，如果本地仓库没有就会从远程仓库下载至本地仓库。

默认本地仓库位置在 ${user.dir}(windows 用户目录)/.m2/repository

Maven学习总结(1)--Maven的安装

• 2017年12月11日

什么是 Maven

Maven是一个项目管理和综合工具。Maven 提供了开发人员构建一个完整的生命周期框架。开发团队可以自动完成项目的基础工具建设，Maven 使用标准的目录结构和默认构建生命周期。

在多个开发团队环境时，Maven 可以设置按标准在非常短的时间里完成配置工作。由于大部分项目的设置都很简单，并且可重复使用，Maven 让开发人员的工作更轻松，同时创建报表，检查，构建和测试自动化设置。

概括地说，Maven 简化和标准化项目建设过程。处理编译，分配，文档，团队协作和其他任务的无缝连接。 Maven 增加可重用性并负责建立相关的任务。

对我们目前来说可以简单的理解为传统项目需要手工导入 jar 包到 lib 目录下，会导致项目文件变大并且不好管理，Mavern 通过仓库的概念实现共享 jar 包，明显的将项目大小变小了。
